---
title: "Environment Variables"
description: "The complete server and application runtime configurations."
---

Configure your Remlo integration across network execution, external ecosystem access, and cryptographic boundaries. **Never expose server-only variables to the client.**

| Variable | Required | Description | Server-Only |
|----------|----------|-------------|-------------|
| `NEXT_PUBLIC_TEMPO_RPC` | Yes | The active Tempo Moderato Node URL (`https://rpc.moderato.tempo.xyz`). | No |
| `NEXT_PUBLIC_TEMPO_CHAIN_ID` | Yes | The Tempo network ID (`42431`). | No |
| `NEXT_PUBLIC_PRIVY_APP_ID` | Yes | The Privy client app ID. | No |
| `NEXT_PUBLIC_SUPABASE_URL` | Yes | The REST endpoint URL for querying structured data. | No |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Yes | The public anon key for basic data ingestion. | No |
| `NEXT_PUBLIC_PAYROLL_TREASURY` | Yes | The active `PayrollTreasury` deployed address on Moderato. | No |
| `NEXT_PUBLIC_PAYROLL_BATCHER` | Yes | The active `PayrollBatcher` deployed address on Moderato. | No |
| `NEXT_PUBLIC_EMPLOYEE_REGISTRY` | Yes | The active `EmployeeRegistry` deployed address on Moderato. | No |
| `NEXT_PUBLIC_STREAM_VESTING` | Yes | The active `StreamVesting` deployed address on Moderato. | No |
| `NEXT_PUBLIC_YIELD_ROUTER` | Yes | The active `YieldRouter` deployed address on Moderato. | No |
| `NEXT_PUBLIC_APP_URL` | Yes | The absolute web client URI (`https://remlo.xyz`). | No |
| `SUPABASE_SERVICE_KEY` | Yes | High-context Database management bypassing row level security policies. | **Yes** |
| `BRIDGE_API_KEY` | Yes | The explicit Bridge identifier to generate and operate the API suite. | **Yes** |
| `BRIDGE_WEBHOOK_SECRET` | Yes | Cryptographic RSA public signature verification from incoming data. | **Yes** |
| `REMLO_TREASURY_ADDRESS` | Yes | Receiving wallet for all completed MPP fractional toll transactions. | **Yes** |
| `REMLO_AGENT_PRIVATE_KEY` | No | Private spending key initializing internal platform-wide testing scripts. | **Yes** |
| `RESEND_API_KEY` | Yes | Configuration to broadcast employee verification and payment receipt emails. | **Yes** |
| `CLAUDE_API_KEY` | Yes | Access tier powering integrated data anomaly sweeps inside runs. | **Yes** |
| `STRIPE_SECRET_KEY` | No | Secondary integration supporting external fiat-backed logic parsing. | **Yes** |
| `MPP_SECRET_KEY` | Yes | Randomized 32-byte base64 secure key constructing MPP payload receipts. | **Yes** |
