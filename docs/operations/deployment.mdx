# Deployment

Remlo has three production surfaces to think about:

- the Next.js application
- the Mintlify docs site
- the Tempo contracts and their public addresses

If any one of those drifts, the product can look healthy while critical flows like treasury, invite, or payroll quietly fail.

## Application deployment

The app is designed for Vercel with `pnpm build` as the production build command.

### App checklist

1. Set every required value from [Environment Variables](environment-variables).
2. Confirm `NEXT_PUBLIC_APP_URL` points to your live domain, not an old preview URL.
3. Confirm the deployed contract addresses match every `NEXT_PUBLIC_*` contract variable.
4. Verify Supabase service credentials are present for protected routes and webhooks.
5. Verify Bridge keys and webhook secret are set before enabling KYC, cards, or bank off-ramp.
6. Verify `REMLO_AGENT_PRIVATE_KEY` and `MPP_SECRET_KEY` before enabling machine-paid write flows.
7. Verify each active employer row has `employer_admin_wallet` populated with the real payroll wallet used for treasury deposits.

## Contract deployment

The Foundry scripts in `contracts/` target Tempo Moderato test infrastructure.

After a contract redeploy:

1. Update the public contract variables in Vercel
2. Update local `.env.local`
3. Verify `/dashboard/settings` still shows the employer wallet as `Ready`
4. Re-test treasury, payroll, and yield against the new addresses

Do not rely on stale fallback addresses after a redeploy. Remlo reads contract state directly from the addresses in its public environment.

## Docs deployment

Mintlify reads `docs.json` from the project root and the MDX pages from `docs/`.

### Docs checklist

1. Connect the project through the Mintlify GitHub app.
2. Keep `docs.json` at the repo root.
3. Keep docs content and docs-only brand assets under `docs/`.
4. Use preview builds for layout or copy review before merge.

## Post-deploy smoke test

| Surface | What to verify |
| --- | --- |
| Public site | `/`, `/pricing`, `/about`, `/contact`, `/press`, `/status`, and legal pages |
| Auth | `/login`, `/register`, `/invite/[token]`, `/kyc/[token]` |
| Employer | `/dashboard`, `/dashboard/settings`, `/dashboard/team`, `/dashboard/payroll/new`, `/dashboard/treasury`, `/dashboard/api-access` |
| Employee | `/portal`, `/portal/payments`, `/portal/card`, `/portal/wallet`, `/portal/settings` |
| Webhooks | Bridge and Tempo signatures verify in the deployed environment |
| MPP | 402 challenge flow works and receipts are recorded for paid endpoints |

## Release gate

Do not call the deployment complete until all of the following are true:

1. `pnpm build` passes
2. `pnpm exec tsc --noEmit` passes
3. Employer wallet status shows `Ready` in `/dashboard/settings`
4. Treasury and yield load without identity errors
5. Invite links resolve on the live domain
6. The MPP pricing page matches the backend charges
7. Bridge and Tempo webhooks accept signed requests
