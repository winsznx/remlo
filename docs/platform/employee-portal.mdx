# Employee Portal

The employee application lives under `/portal`. It is designed so a worker can receive payroll, understand what was paid, and move funds without needing to understand the underlying chain stack.

## What employees can do

| Area | Outcome |
| --- | --- |
| Home | See balance, most recent payroll, and quick actions |
| Payments | Review payroll history, memo context, and transaction status |
| Card | View card state and initiate spend-related actions where enabled |
| Wallet | Confirm which wallet is linked to payroll |
| Settings | Update profile and off-ramp preferences |

## Route map

| Route | Purpose |
| --- | --- |
| `/portal` | Balance overview and quick actions |
| `/portal/payments` | Payment history with memo details |
| `/portal/card` | Card visibility and bank transfer entry point |
| `/portal/card/activate` | Card activation handoff |
| `/portal/wallet` | Wallet identity and sync state |
| `/portal/settings` | Profile and account settings |
| `/portal/settings/offramp` | Bank account and off-ramp preferences |

## Salary experience

Remlo supports two employee payout views:

- standard payroll, where the employee sees completed payroll transfers
- streaming payroll, where the employee sees accrued salary that can be claimed

The portal abstracts both into one balance-first interface.

## Payment history

Each payment can surface:

- amount
- status
- decoded memo context
- transaction hash
- explorer link
- settlement timing when available

That gives employees an understandable audit trail instead of a raw token transfer list.

## Cards and bank off-ramp

Where Bridge is enabled, the portal can expose:

- card status and activation
- bank off-ramp preferences
- direct employee off-ramp actions

Those actions are available through first-party employee routes, while the MPP off-ramp endpoint remains available for automation.

## Wallet and identity

The wallet page is critical during onboarding. It shows whether the employee’s current Privy wallet and stored payroll wallet line up, which helps catch invite-claim or account-mismatch issues early.
