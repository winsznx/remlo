# Employer Dashboard

The employer application lives under `/dashboard`. It is where finance, ops, and compliance teams manage payroll from funding through settlement.

## Core modules

| Module | What it controls |
| --- | --- |
| Overview | Treasury, payroll, headcount, and yield summaries |
| Team | Employee records, invites, compensation, payment holds, and KYC readiness |
| Payroll | Drafts, run creation, execution, and run history |
| Treasury | Available balance, locked payroll, deposits, and yield context |
| Compliance | KYC state, manual review, and policy-related events |
| Cards | Employer-managed spend controls for employee card programs |
| API Access | MPP pricing, sessions, receipts, and the demo agent |
| Settings | Canonical employer wallet, organization profile, and billing |

## The four employer loops

### 1. Onboard the team

Create employees, issue invites, open KYC flows, and confirm each worker is payout-ready.

### 2. Keep treasury healthy

Confirm the employer wallet is synced, fund treasury, and monitor available versus locked payroll balance.

### 3. Prepare and submit payroll

Build a payroll run, review memo and compliance checks, then submit the settlement transaction.

### 4. Monitor outcomes

Review payment history, holds, compliance events, receipts, and API usage from the same workspace.

## Important operator behaviors

- Employee payment state is now explicit. After you pause payments, the next action becomes **Resume payments**, not another pause.
- Treasury, yield, and payroll reads resolve from the employer’s canonical wallet. If the wallet is not synced, these views should be treated as blocked rather than guessed.
- API Access is not a marketing page. It is the operator console for paid machine workflows.
