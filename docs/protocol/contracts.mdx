Remlo's execution layer relies on a set of audited smart contracts deployed on the Tempo network. All operations, from holding employer funds to distributing payroll, are handled via these contracts.

### Moderato Testnet Deployments

| Contract | Address | Purpose |
|----------|---------|---------|
| **PayrollTreasury** | `0xeFac4A0cC3D54903746e811f6cd45DD7F43A43a5` | Holds employer stablecoin balances and funds gas logic. All payroll runs pull from this vault. |
| **PayrollBatcher** | `0x90657d3F18abaB8B1b105779601644dF7ce4ee65` | Executes parallel stablecoin transfers to multiple employee wallets in a single transaction. |
| **EmployeeRegistry** | `0xe7DdA49d250e014769F5d2C840146626Bf153BC4` | Maps employee wallets to their employers and verifies TIP-403 compliance policies during registration. |
| **StreamVesting** | `0x83ac4D8E7957F9DCD2e18F22EbD8b83c2BDD3021` | Manages real-time salary streaming, allowing employees to claim accrued balances before payday. |
| **YieldRouter** | `0x78B0548c7bb5B51135BBC87382f131d85abf1061` | Optimizes idle treasury funds across DeFi protocols, generating yield for employers and employees. |

When interacting with the system programmatically, ensure you are referencing the correct contract addresses and the network is explicitly set to Tempo Moderato (Chain ID: `42431`).
