# Privy Integration

Privy is the identity and wallet layer for Remlo. It keeps wallet creation out of the critical path so employers and employees can use the product without first becoming chain power users.

## Embedded wallet behavior

`embeddedWallets.createOnLogin` is set to `users-without-wallets`, which means:

1. a new employer can start without bringing a wallet
2. an employee can claim an invite without any chain-specific setup
3. a self-custodied wallet can still be linked later

## How role resolution works

- Privy proves identity
- Supabase decides whether that identity is an employer, employee, or admin
- middleware routes the user into `/dashboard`, `/portal`, or `/admin`

Wallet identity is therefore necessary, but not sufficient, to determine workspace access.

## WalletConnect

WalletConnect can be enabled when `NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID` is present. That makes external-wallet login available without changing the embedded-wallet onboarding path.
